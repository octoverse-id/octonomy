## Non-negotiables (every phase)

- **Global = `(namespace_type NULL, namespace_id NULL)`**, mirroring `application_id IS NULL`.
  No `'global'` string sentinel; the literal `'global'` is reserved/rejected.
- **Django-native conditional `UniqueConstraint`s + `CheckConstraint` only.** NO raw
  Postgres DDL, NO `CREATE INDEX CONCURRENTLY` — breaks the SQLite+Postgres CI and the
  migration-drift gate. (Accept a brief maintenance window for the constraint swap; see NS-6.)
- **TDD, `cov-fail-under=90` stays green in each PR.** Tests land with the code they cover.
- **Fail-closed grants:** a NULL-namespace grant reaches global rows only. Merchant access
  requires an explicit grant or the explicit wildcard marker. NULL is NOT a namespace wildcard.
- **v1 stays global-only and unchanged**; v1 rejects `X-Namespace-*` with a named 400.

## Phase landmines (from the eng review — wire these in, with tests)

- **#39:** design the explicit **wildcard grant marker** (sentinel value or boolean),
  collision-proof vs caller strings, rejected in data rows. Split `tag_assignments`'
  single plain `UniqueConstraint` (`assignments/models.py:51-54`) into conditional
  constraints — 2 shapes only (`app SET ns NULL` / `app SET ns SET`), since
  `application_id` is NOT NULL there. Backfill grants to global; do NOT auto-promote.
- **#40:** one `grant_authorizes(grant, tenant, app, scope)` predicate. The
  `if tenant_wide: return True` early-return at `core/auth.py:103-107` must NOT bypass
  the namespace gate — it's the skeleton-key vector. `include_global` needs an
  authorized-scope-set, not a request-level boolean. **REG-1** (tenant-wide grant +
  merchant request → DENY) and **REG-2** (legacy NULL grant → global ALLOW / merchant
  DENY) are mandatory.
- **#42:** build `ScopeContext` in the DRF version-aware layer, not `RequestContextMiddleware`
  (it runs before version resolution). Middleware reads `request.scope_context`
  post-response to add namespace to the `request_completed` log.
  **Detail-by-UUID leak (P1):** every object-by-UUID lookup (`GET /api/v1/tags/{id}`,
  aliases, vocab, audit) needs an explicit scope filter — v1 → `namespace IS NULL`,
  v2 → request scope, 404 on mismatch. The isolation sweep must cover detail endpoints,
  not just lists.
- **#41:** add the missing `IntegrityError` guard to `bulk_assign_tags` (only single
  `assign_tag` has it). Make `usage_count` namespace-scoped (filtered `Count`,
  `tags/selectors.py:9,17`) + a side-channel test (merchant_a count excludes merchant_b).
- **#43:** carry namespace on `AuditContext` so all ~10 audit/outbox call sites propagate
  it with no per-site args. A missed site emits a NULL-namespace event = leak.
